# IPL 2025 Performance Analytics

## Overview
This project provides an in-depth, interactive analysis of player performances for IPL 2025. It features a complete data pipeline, starting from data extraction and transformation using Python, through to complex data modeling and visualization in Power BI. 

## Dataset Source
The raw ball-by-ball and match data used for this analysis was sourced from [Cricsheet](https://cricsheet.org/). which consisted of 17000+ deliveries fot the 2025 IPL tournament The dataset was then parsed, cleaned, and transformed using Python (`IPL_2025.ipynb`) before being loaded into Power BI for modeling.

## Repository Contents
* **`IPL_2025.ipynb`**: The Jupyter Notebook containing the data extraction, cleaning, and transformation logic using Pandas.
* **`Main and Norm Tables`**: The cleaned datasets generated by the Python script, ready for ingestion.
* **`IPL 2025.pbix`**: The fully interactive Power BI report containing the data model, DAX measures, and dashboard visuals.

## DAX Calculations & Logic

This section outlines some of the Data Analysis Expressions (DAX) used to power the dashboard.

**Total Runs**
Calculates the aggregate runs scored across the tournament or by a selected team.
```dax
Total Runs = SUM(Deliveries[Runs_Total])
```

**Total Wickets**
Counts the total number of dismissals, excluding run-outs where the bowler does not get credit.
```dax
Total Wickets = 
CALCULATE(
    SUM(Deliveries[Is_Wicket]),
    Deliveries[Dismissal_Kind] IN {
        "caught", "bowled", "lbw", "stumped", "caught and bowled", "hit wicket"
    }
)
```

**Total Fours & Sixes**
Counts the number of boundaries hit.
```dax
Total Fours = CALCULATE(COUNTROWS(Deliveries), Deliveries[Runs_Batter] = 4)
Total Sixes = CALCULATE(COUNTROWS(Deliveries), Deliveries[Runs_Batter] = 6)
```

**Player Wickets Taken**
Bowler's wickets to display accurately on their individual profile card removing Run Outs
```dax
Player Wickets Taken = 
CALCULATE(
    SUM(Deliveries[Is_Wicket]),
    Deliveries[Dismissal_Kind] IN {
        "caught", "bowled", "lbw", "stumped", "caught and bowled", "hit wicket"
    },
    USERELATIONSHIP(Players[Player_ID], Deliveries[Bowler_ID]),
    REMOVEFILTERS('Teams') 
)
```

**Player Economy Rate**
Calculates runs conceded per over bowled. By evaluating variables locally, it isolates runs strictly attributed to the bowler (excluding Byes/Leg-Byes) and divides by legal deliveries only.
```dax
Player Economy Rate = 
VAR RunsConceded = 
    CALCULATE(
        SUMX(
            Deliveries, 
            Deliveries[Runs_Total] - Deliveries[Byes] - Deliveries[LegByes]
        ),
        USERELATIONSHIP(Players[Player_ID], Deliveries[Bowler_ID]),
        REMOVEFILTERS('Teams')
    )

VAR LegalDeliveries = 
    CALCULATE(
        COUNTROWS(Deliveries), 
        Deliveries[Is_Wide] = 0,
        Deliveries[Is_NoBall] = 0,
        USERELATIONSHIP(Players[Player_ID], Deliveries[Bowler_ID]),
        REMOVEFILTERS('Teams')
    )

VAR OversBowled = LegalDeliveries / 6

RETURN 
    DIVIDE(RunsConceded, OversBowled, BLANK())
```

**Player Batting Strike Rate**
Crucially, it filters out Wides (`Is_Wide = 0`) from the `BallsFaced` denominator, as wides do not count as deliveries faced by the batsman according to standard cricket rules.
```dax
Player Strike Rate = 
VAR TotalRuns = 
    CALCULATE(
        SUM(Deliveries[Runs_Batter]),
        USERELATIONSHIP(Players[Player_ID], Deliveries[Batter_ID]),
        REMOVEFILTERS('Teams')
    )

VAR BallsFaced = 
    CALCULATE(
        COUNTROWS(Deliveries),
        Deliveries[Is_Wide] = 0, 
        USERELATIONSHIP(Players[Player_ID], Deliveries[Batter_ID]),
        REMOVEFILTERS('Teams')
    )

RETURN 
    DIVIDE(TotalRuns, BallsFaced, BLANK()) * 100
```
